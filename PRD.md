# PRD: Attendance Management System - Martial Arts Academy

## 📋 **1. מטרה ובעיה**

### **הבעיה:**
מאמן אומניות לחימה כיום מנהל יומן נוכחות ידני/טבלה שלא מאורגנת, קשה לעקוב אחריה, ואין דרך קלה לנהל קבוצות שונות, סטטוס הרשמה, וצפייה בנתונים בהיסטוריה.

### **הפתרון:**
יישום ווב מקומי (LocalStorage) לניהול מלא של:
- סימון נוכחות יומי לפי קבוצות
- ניהול חניכים עם מטא-דאטה (שם, דרגה, גיל, סטטוס הרשמה, תאריך התחלה)
- דוחות חודשיים
- ייבוא/ייצוא נתונים

---

## 👥 **2. משתמש יעד ותרחיש שימוש**

### **משתמש יעד:**
מאמן/י אומניות לחימה המנהלים קבוצות מתאימות, צריכים לעקוב אחרי נוכחות, להרשום חניכים חדשים, ולראות דוחות היסטוריים.

### **תרחיש שימוש:**
1. **בתחילת כל שיעור:** מאמן פותח את האפליקציה, בוחר את הקבוצה, מסמן נוכחות (✓ נוכח, ✗ לא הגיע)
2. **חניך חדש מגיע:** מוסיפים חניך חדש עם פרטים (שם, דרגה, גיל), אוטומטי מזומנת התאריך ההנוכחות על סטטוס "Pending"
3. **אישור הרשמה:** עמוד נפרד מראה חניכים "Pending" - לחיצת ✓ = הופך לרשום (Registered)
4. **צפייה בנתונים:** בסוף החודש, המאמן צופה בטבלת הנוכחות של כל החניכים לחודש
5. **תחילת שנה:** ב-1.09 הנתונים מתאפסים, אבל אפשר לטעון חניכים מהשנה הקודמת

## 🎯 **3. היקף (Scope)**

### **✅ In Scope (נכנס לפרויקט):**

#### **עמוד 1: Daily Attendance**
- הצגת tabs לקבוצות (custom names)
- **תוספת קבוצה חדשה** + עריכה + מחיקה
- **תוספת חניך חדש** עם:
  - שם + משפחה
  - דרגה
  - גיל
  - קבוצה (בחירה מ-dropdown)
  - סטטוס הרשמה (toggle: Registered/Pending)
  - תאריך התחלה (auto - היום)
- **סימון נוכחות** ליום זה עבור כל חניך בקבוצה
  - Checkbox / V/X

#### **עמוד 2: Pending Students**
- רשימה של **רק** חניכים עם סטטוס Pending (חדשים שלא נרשמו)
- כל חניך: שם, תאריך התחלה, קבוצה
- כפתורים:
  - **✅** → הופך ל-Registered
  - **❌** → מחיקה מוחלטת מהDB

#### **עמוד 3: Monthly Report**
- Tabs קבוצות
- טבלה: שורה = חניך, עמודה = יום החודש
- סטטוס בכל תא: ✓ (נוכח), ✗ (לא הגיע), - (ריק)
- כל חניך עם tag: 🟢 Registered / 🔴 Pending

#### **דאטה וניהול:**
- **תאריך התחלה אוטומטי** - כשמוסיפים חניך, שומרים את היום
- **ב-1.09:** Reset שנתי - מנקים את הנתונים היומיים אבל שומרים את החניכים
- **Import/Export:**
  - ייצוא לצורת JSON / CSV
  - ייבוא מקודם (upload קובץ)
- **LocalStorage:** כל הנתונים ברמה הלוקלית של הדפדפן

#### **Features נוספות:**
- עריכת חניך (שם, דרגה, גיל, קבוצה)
- מחיקת חניך
- עריכת שם קבוצה
- מחיקת קבוצה (+ warning)

### **❌ Out of Scope (לא בפרויקט זה):**

- Multi-user / login system
- Cloud sync / backend server
- Mobile app native (רק responsive web)
- Analytics / graphs מתקדמות
- Notifications / reminders
- Payment / subscription

## 🛠️ **5. בחירות טכניות**

### **Stack:**
- **Frontend:** HTML5 + CSS3 + JavaScript (Vanilla)
- **Storage:** Browser LocalStorage (JSON)
- **Data Format:** JSON (internal), CSV (export)

### **אילוצים:**
- רק דפדפן (Chrome, Firefox, Safari)
- כל נתון מקומי, אין backend
- Responsive (עבוד גם בנייד)
